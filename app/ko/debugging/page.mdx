# 디버깅 (개발 전용)

상세 캐시 로그 활성화:
```bash path=null start=null
# .env.local
NEXT_PUBLIC_NEXUS_DEBUG=true
```

## 요청 이벤트
- START — 요청 시작
- SUCCESS — 성공 완료
- ERROR — 실패
- TIMEOUT — 타임아웃으로 중단

## 캐시 이벤트
- HIT/MISS — 클라이언트 캐시 조회 결과
- SKIP/MATCH — 서버가 notModified로 하이드레이션을 생략하거나 클라이언트 엔트리와 일치
- DELEGATE — `<NexusSuspense>`로 서버가 렌더링을 위임
- SET/UPDATE/DELETE — 클라이언트 캐시 쓰기 동작
- 개발 로그에 baseKey, tags, ttl, revalidate, size/maxSize 표시

> 로그는 개발 환경에서만 출력됩니다.
